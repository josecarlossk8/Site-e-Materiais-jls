# Central JLS

Esta pasta organiza o processo de estrategia, conteudo, comercial e operacao da JLS Log dentro do Codex.

A regra principal: antes de criar qualquer material novo, consulte primeiro os arquivos de `00_contexto_master`. Eles guardam o posicionamento, tom de voz, nichos, ofertas, provas de confianca e dados operacionais que nao podem se perder.

## Como usar

0. Para saber como dividir conversas e projetos dentro do Codex, comece por `GUIA_DE_USO_NO_CODEX.md`.
1. Para site, PDFs e materiais ja publicados, use `01_site_e_materiais`.
2. Para abordagem comercial, ICP, objeções e playbook, use `02_estrategia_comercial`.
3. Para posts, artigos, campanhas e calendario editorial, use `03_conteudo_marketing`.
4. Para prospeccao ativa, cadencias e listas de contas, use `04_prospeccao`.
5. Para organizar produtos logisticos, SLA, fluxos e escopo de atendimento, use `05_operacao_e_produtos`.
6. Para provas, cases, chancelas e evidencias, use `06_cases_e_provas`.
7. Para criar ou melhorar Google Perfil da Empresa, use `07_google_perfil_empresa`.
8. Para campanhas, funil e rotina comercial, use `08_campanhas_rotina_comercial`.

## Fluxo recomendado no Codex

- Toda nova conversa sobre estrategia deve começar com o contexto master.
- Toda nova peça de conteudo deve indicar o nicho, objetivo e CTA.
- Toda abordagem comercial deve ser salva com versao, publico e canal.
- Toda nova prova ou case deve entrar em `06_cases_e_provas` antes de virar copy.
